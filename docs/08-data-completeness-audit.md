# 데이터 완전성 감사 (Data Completeness Audit)

작성일: 2026-06-18 · 대상: `C:\purchasesystem` 전 도메인 · 비교기준: DIPS(dwhf2018)/DFS(dwfb2020) 참고시스템 + 구매/회계/품질 실무표준

> 목적: "프로세스가 도는가"가 아니라 **각 테이블/서비스가 실무에서 실제로 필요한 정보를 빠짐없이 보유하는가**를 항목 단위로 점검. 6개 클러스터를 병렬 감사한 결과의 종합본.

---

## 0. 핵심 결론 — 횡단(Cross-cutting) 갭 Top 8

개별 도메인보다 **여러 도메인을 동시에 관통하는 구조적 결손**이 더 치명적이다. 우선순위는 이 횡단 갭부터.

| # | 횡단 갭 | 영향 도메인 | 현상 | 심각도 |
|---|---|---|---|---|
| C1 | **과세구분(TAX_TYP) 부재 + VAT 10% 하드코딩** | PR·RX·AU·PO·GR·CL·PA·세금계산서·전표 전체 | 면세(농수산물)·영세율·불공제를 표현 못 함. 식품사인데 면세 원재료 처리 불가. 모든 서비스가 `×0.1` 고정 | ★★★ |
| C2 | **식품 추적성(LOT/제조일자/유통기한) 전무** | GR·QL·QC·CLM | 입고/검사/부적합/클레임 어디에도 LOT·유통기한 없음 → 회수(리콜)·HACCP·선입선출 불가. 동원 식품사 특성상 최우선 | ★★★ |
| C3 | **마스터 값 거래시점 미스냅샷** | CL·PO·세금계산서·전표 | 결제조건·통화·환율·은행계좌·사업자정보가 마스터(VD_VENDOR/계약)에만 있고 거래문서에 박제 안 됨 → 마스터 변경 시 과거 전표 소급 변질 (Hfrnd 원료단가 박제 이슈와 동일 패턴) | ★★★ |
| C4 | **첨부파일(CM_ATTACH) FK 연계 부재** | 거의 전 도메인 | attach 도메인은 존재하나 협력사·계약·PR·견적·입고·품질·클레임이 실제로 파일을 매달지 못함. 사업자등록증·통장사본·CoA·계약서 보관 불가 | ★★★ |
| C5 | **납품지/입고창고(WH_CD·배송지) 부재** | PO·GR | 어디로 배송/입고되는지 미기록 → 재고·물류 연계 불가. 참고시스템은 RD_LOCAT/WH_CD 보유 | ★★ |
| C6 | **결제조건 코드화 + PO 자동전파 부재** | VD·PO·CL·지급 | PAY_COND가 자유텍스트 1개. 지급주기/방법/어음 구분 없음. 협력사 기본 결제조건이 발주·정산에 전파 안 됨 → 지급예정일 자동계산 불가 | ★★ |
| C7 | **지급(Payment) 단계 통째 부재 + 회계전표 단일라인** | 정산·전표·지급 | 정산→세금계산서→전표까지만 있고 "언제·얼마·어떻게 지급"이 없음. AC_SLIP은 단일 DR/CR + 계정"명" 한글 하드코딩 → 복식부기 다라인·부가세대급금 분리·계정코드·ERP전표번호 회수 불가 | ★★ |
| C8 | **요청납기 vs 확정납기 미분리 / 분할납품 일정 부재** | PR·PO·GR | 납기가 단일 컬럼. 희망·확정 구분 없고 다회차 납품 모델 없음 | ★★ |

추가 횡단: **예산통제 전무**(PR·구매계획 — 예산코드/잔액검증 없음), **유찰/재공고·분할선정 상태 부재**(RX·AU), **거래중지/해지 사유·이력 부재**(VD·계약).

## 1. 협력사(Vendor)

**잘 갖춤**: 은행/계좌/예금주, 통화, 세금계산서 발행유형, 산업부문/거래형태/규모/세그먼트, 담당자/면허 서브테이블, 평가(세그먼트별 등급·2단계 가중), 신규등록 심사 워크플로우.

**보강 필요 (H)**
- `PAY_TERM_CD`/`PAY_METHOD_CD`/어음여부·만기일수 — 결제조건 코드화(현재 PAY_COND 자유텍스트 1개)
- `TAX_TYP`/`WHT_YN`(과세형태·원천징수) — 용역·프리랜서 3.3% 처리
- `CREDIT_LIMIT`/`TRADE_LIMIT_AMT`(여신·거래한도) — 한도초과 발주 통제
- `STOP_YMD`/`STOP_REASON` + 거래중지/재개 비즈니스 메서드·이력 (현재 STOP 상태만, 사유·일자 없음)
- 협력사 **첨부**(사업자등록증/통장사본/인증서, 필수여부) — 계좌검증·세금계산서 전제
- 평가·인증 **유효기간 만료관리**(VALID_TO/NEXT_EVAL_YMD/ALARM_BEFORE_DAY) — 적격성 라이프사이클

**보강 필요 (M)**: 업태/업종 분리, 우대분류(중소/여성/장애인기업), 품질담당자(QULT_CHRG_ID), 임직원수/매출/신용등급, 담당 구매조직, 세금계산서 수신 이메일, 감시결과(RISK)의 마스터 환류·시정조치 기한추적.

**프로세스 갭**: 결제조건이 PO로 자동 전파 안 됨 / 거래중지 사유 미기록 / 감시(RISK) 결과가 발주 차단에 미연결.

## 2. 품목·단가·계약 (Item / CT / IP / BC)

**잘 갖춤**: IP 비중첩 구간분할 엔진·이력, BC 개정/자동갱신/만료알림/조항, CT 통화·총액·발주량범위·적용기간.

**보강 필요 (H, 식품 핵심)**
- IT_ITEM: `STOCK_UNIT_CD`+환산계수(구매단위≠재고단위), `STORAGE_COND`(냉장/냉동/상온), `ORIGIN_CD`(원산지), `SHELF_LIFE_DAYS`/`EXP_MGMT_YN`(유통기한 관리), `ACCT_CD`(계정과목), `BARCODE/GTIN`
- CT/BC: 계약서 **첨부**+전자서명(`*_FILE_NO`/`ESIGN_YN`), 납품조건/인도조건(Incoterms), CT 결제조건(헤더), CT 만료알림(BC엔 있음)
- IP/CT: `PLT_CD`(공장 차원) — 다공장 단가, 단가 적용 **우선순위**(PRICE_PRIORITY)

**보강 필요 (M)**: HS코드, 제조사/브랜드, QC필요여부/기준, 알러지/원료, 안전재고/MOQ/리드타임, 대표협력사·대표단가, 기준수량(PER "단위당"), 지체상금율, 품질보증조건, 해지사유/일자.

**프로세스 갭**: 단위환산 부재로 구매↔입고 수량/금액 환산 불가 / `ContractService.applyAmount`가 수량 미반영 단가합산(검토) / 단가 우선순위 규칙 없음(비결정적 선택) / 식품 규제데이터(원산지·보관·유통기한) 동적속성 의존.

## 3. 구매요청·견적·역경매 (PR / RX / AU)

**잘 갖춤**: 채번·상태머신·소스추적, AU 틱사이즈/자동연장/입찰이력/순위, RX 평가위원·가중치.

**보강 필요 (H)**
- PR: `PLANT_CD`(사업장)/`WH_CD`(창고), `BUDGET_CD`/`BUDGET_AMT`+예산검증, `URGENT_YN`/사유, `TAX_TYP`, **첨부**, 확정납기 분리
- RX: 마감 **일시(TIMESTAMP)**(현재 DATE라 시각마감 불가), 결제·납기조건, 견적유효기간, **첨부**, 환율
- AU: 입찰 **시작일시**, **예가/낙찰하한**(현재 상한만), 익명입찰 정책

**보강 필요 (M)**: 계정과목/코스트센터/WBS, 승인금액(부분승인), 과세/세액 분리, 유찰/재공고 상태, 분할선정(품목별 협력사) 구조.

**프로세스 갭**: 예산통제 전무 / 첨부 인프라 미연계 / 전 클러스터 공급가만 계산(세액·총액 미분리) / 날짜타입 비대칭(시각마감 불가) / PR 분할소싱 잔량추적 없음.

## 4. 발주·입고 (PO / GR)

**잘 갖춤**: PO 분할결제(PO_PAY_PLAN)/발주유형/보증금/개정차수, GR 검수 합격/불합격 수량분리·입고누계·입고취소 시 PO누계 차감.

**보강 필요 (H)**
- PO: 납품지/배송주소(`DLV_ADDR`/`WH_CD`), 환율, `TAX_TYP`, 요청납기 vs 확정납기, 협력사 전송/납품확약(`VD_SEND_YN`/`VD_ACK_YN`)
- PO_ITEM: 품목별 납품장소(RD_LOCAT), 과세/면세, **분할납품 일정**(PO_DLV_SCHEDULE 신설)
- GR: 입고창고/로케이션, 거래명세서번호, 세금계산서 매칭(`TAX_INV_NO`/3-way), 운송정보
- GR_ITEM: **LOT_NO/제조일자/유통기한**(식품), 반품수량(`RTN_QTY`)/사유, 시험성적서번호

**보강 필요 (M)**: 인도조건 코드, 운송비, 계정/코스트센터, 개정사유, GR 납기지연 패널티(지체상금), 입고유형(정상/반품/무상).

**프로세스 갭**: VAT 10% 하드코딩 / 환율 미보유로 외화발주 환산불가 / 발주확정 후 협력사 통보·확약 이벤트 없음 / 반품경로 별도 없음(취소와 혼재) / 식품 추적성 전무.

## 5. 정산·지급 (CL / 갭정산 / PA / AV / 세금계산서 / 전표)

**잘 갖춤**: 갭정산 조정율 엔진, PA 매입조정·정산연계, AV 선급금/반제 잔액관리, 세금계산서/전표 시뮬 발행·전송 골격.

**보강 필요 (H)**
- CL: 정산대상기간(FROM/TO, 현재 월만), 전기일자/회계기간(ACC_DATE/ACC_YM), 지급예정일(결제조건 만기), 결제조건 스냅샷, 마감잠금/해제 **이력**(누가·언제·왜)
- 세금계산서: **국세청 승인번호**, 작성일자/발행일시 분리, **공급자·공급받는자 사업자정보**(법정기재), 영세율 구분, **수정세금계산서**(원본/사유), **품목명세 라인테이블**(현재 없음)
- 전표: **AC_SLIP_LINE 자식테이블**(복식부기 다라인), 계정과목**코드**, 부가세대급금 분리, 회계기간, ERP전표번호 회수, 역분개
- 신설: **지급(PY_PAYMENT) 테이블** — 지급예정/실행/방법/어음/계좌/지급전표

**보강 필요 (M)**: 통화/환율, 원천징수액, 선급금 상계 표면표기, 조정 부가세/계정, PA 정산 역링크(이중반영 방지), AV 지급수단/실지급일/계약연계.

**프로세스 갭**: 지급단계 통째 부재 / 마스터 미스냅샷 / 부가세 일관성 붕괴(정산 NET vs 세금계산서 VAT) / 전표 계정명 하드코딩·공급가+세액 단일금액 / SAP 회신 전표번호 미저장.

## 6. 품질·클레임·계획 (QL / QC / CLM / PP)

**잘 갖춤**: CAPA 시정/예방·근본원인·담당/기한/완료/검증상태, 검사 합부 자동판정 골격, 클레임 상태흐름·협력사통보, 구매계획-실적(금액) 대비.

**보강 필요 (H)**
- QL: `LOT_NO`/제조일/유통기한, 불합격처리(`DISPOSITION` 반품/특채/폐기), 불합격수량, CoA 첨부, 샘플링수량
- QL_RESULT: 규격 **상·하한**(현재 문자열 단일값→자동판정 불가), 측정값 **수치화**, 시험분류(미생물/관능/이물/잔류농약)
- QC_ACTION: 효과검증 데이터(검증자/검증일/검증결과)
- CLM: 보상방식(반품/교환/감액/현금), 클레임수량, 실회수금액(청구액과 분리)
- PP: **월별 분해**(M01~M12), 실적**수량**(현재 금액만), 계획 **변경이력/버전**

**보강 필요 (M)**: 발생원인분류·불량률·손실금액·클레임연계(NONCONF), 협력사 회신/귀책인정(CLM), 계획 협력사/단가/예산연계(PP).

**프로세스 갭**: QL불합격→QC부적합→CLM클레임 자동연계 없음(텍스트로만 느슨히) / 규격·측정값 문자열이라 통계·SPC 불가 / CAPA 기한경과 에스컬레이션 없음 / 클레임 금전 단일컬럼 / 계획 예산 미연계.

## 7. 권장 구현 로드맵 (우선순위)

**1차 — 횡단 기반(가장 높은 레버리지)**
1. **과세구분 체계(C1)**: CM_CODE TAX_TYP(과세/면세/영세/불공제) + IT_ITEM·PR/PO/GR/CL 라인에 TAX_TYP + 서비스 VAT 계산을 품목 과세구분 기반으로 (식품 면세 필수)
2. **식품 추적성(C2)**: GR_ITEM·QL·QC·CLM에 LOT_NO/제조일자/유통기한
3. **첨부 FK 연계(C4)**: 각 문서 화면에서 CM_ATTACH refTyp/refId 연결 활성화(협력사·계약·PR·견적·입고·품질·클레임)
4. **마스터 스냅샷(C3)** + **결제조건 코드화·전파(C6)**: 협력사 결제조건/통화/계좌 → PO·CL 박제, 지급예정일 자동계산

**2차 — 도메인 심화**
5. 발주/입고 납품지·창고·납기분리·분할납품(C5/C8)
6. 정산·전표·지급 회계 정합(C7): AC_SLIP_LINE 다라인 + 지급테이블 + 세금계산서 법정항목
7. 품질 규격 정량화(상하한+수치) + QL→QC→CLM 자동연계

**3차 — 통제/관리**
8. 예산통제(PR·계획), 여신한도(협력사·발주), 유찰/재공고·분할선정, 거래중지 이력, CAPA 에스컬레이션
