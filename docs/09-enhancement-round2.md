# 고도화 라운드2 — DIPS/DFS 기준 5영역 분석

작성일: 2026-06 · 방법: 5개 클러스터 병렬 분석(참고시스템 eMRO SmartSuite9 mapper/VO/UI 조회, 읽기전용) · 우리 현황 대비 미구현·심화 후보 도출

> 1차 데이터 고도화(docs/08, C1~C8)가 "필드/데이터 완전성"이었다면, 라운드2는 **참고시스템의 "워크플로우 메커니즘"과 심화 기능**을 가져오는 것. 5개 시스템 맵 영역(①~⑤)별로 분석.

---

## 0. 횡단 테마 (5영역 공통으로 반복 발견된 4대 축)

| 테마 | 내용 | 해당 영역 |
|---|---|---|
| **A. 변경/취소/시정 폐루프(워크플로우)** | 발주변경·입고변경·마감취소·CAPA·실사를 "정식 상태머신 + 결재 + 이전판/음수전표"로 | ①②③④ |
| **B. 식품 특화** | 의제매입세액공제, 검사로트·특채·사용결정, 원유/원료 공인검사, AEO 공급망안전 | ②③④ |
| **C. 공사·용역 조달** | 선금/기성/잔금 + 보증금 4종 + 기성청구(AR) + 완료기준 분기(수량 vs 기성금액) | ①② |
| **D. 협력사 셀프서비스** | 자가 가입신청, CAPA 회신, 발주확인/납품확약, AEO 자가진단 응답 | ①③④ |
| **E. 플랫폼 성숙도** | 통합 알림센터, 위임전결규정, 감사로그, 배치 스케줄러, 데이터레벨 권한/SSO | ⑤ |

## ① 핵심 구매 사이클 (PR/소싱/PO/GR/재고/MRP)
1. **발주 변경관리(개정) 정식 워크플로우** — 변경요청 상태머신(MOD_REQ_PROG_STS) + 현재판 토글(CRNT_PO_YN) + 이전판 종료/신판 atomic + 차수 delta(수량·단가·납기·장소) 비교 [H/대]
2. **입고변경(GrChg) 사후조정 + 3-way 매칭** — 입고확정 후 가격/수량 BF→AF 조정(협력사/자사 발의 구분) + 발주-입고-세금계산서 정합검증 [H/대]
3. **소싱 심화 3종** — 예가(TGT_PRICE) + 가격팩터 원가구조 비교(UP_FACT_CD 계층) + 재견적/2차입찰(RFX_REV) [H/중~대]
4. **공사·용역 발주 풀스택** — 선금/기성/잔금률 + 보증금 4종(선급/이행/하자/유지) + 기성청구 AR + 완료기준 분기 [H/대]
5. **단가계약 자동발주(AUTO_PO) + 협력사 발주확인/납품확약 포털** [H/중]
- 우리 우위(eMRO 미보유): LOT/유통기한·재고/수불·MRP — 다단계 소요전개·예측소요로 더 밀면 추월

## ② 정산·회계·지급
1. **마감취소→재마감 정규화** — 마감취소헤더 + 취소회차 + 음수전표 + 상태(Q/D/E) [상]
2. **의제매입세액공제** — FICT_VAT_AMT(세코드 IX/IZ) GR→정산→세금계산서 가산 (식품 필수) [상]
3. **지급 3대 미구현** — 어음(발행/만기/결제)·원천징수(WHT)·채권채무 상계(OFFSET) [상]
4. **미지급금 AP 에이징 + 지급마감일정 캘린더** [상]
5. **IF 송수신 전이력 + 재전송 audit 테이블** [상]
- 보너스: 장려금 전표유형 8종(P108) 분화로 회계분리 정확도 ↑

## ③ 협력사·품목·계약
1. **협력사 실사(Audit) 시정조치 폐루프** — 부적합→개선계획→협력사 조치제출(증거첨부·기한)→재실사→효과검증 (DFS auditnew CSRIPHD/DT) [대]
2. **AEO/공급망안전 자가진단 평가 모듈** — 질의서+재무+매출 가산→등급·고위험판정 + 협력사 셀프응답(서약서) (DFS aeo) [대, 식품/수출입]
3. **평가 자동주기 스케줄러 + 이의신청 폐루프** (DFS EvalScheduler·SrmEvalCmpl) [중]
4. **단가 시장가(지수) 연동 + 인상승인 워크플로우** (DFS itemprice GRK_MK_PRICE·CSPIFMD) [중]
5. **전자계약 전자서명(공인인증)** (DFS CrossCert·ESECRMR) [대]
- 추월 포인트(DFS도 없음): 품목 대체품·BOM, 규제↔품질리스크 자동연계

## ④ 품질·관리/통제
1. **검사로트 + 사용결정(UD) + 특채** — 자동합부 → 검사원 최종 사용결정(합격/특채/반품) 분리 (DFS QM-0003) [상, 식품]
2. **협력사 셀프-CAPA 회신 포털 + 자사 심사 게이트** — 협력사가 원인분석/시정/재발방지 직접 입력·제출→자사 판정·승인 (DFS QM-0013) [대]
3. **품질 KPI 환류** — 납기준수율(OTD)·불량률(PPM)·부적합 파레토 → 평가 환류 (참고시스템도 raw만 → 차별화) [상]
4. **원유/원료 공인기관 검사결과** — 체세포·세균수·이화학 → 등급제·정산 인센티브 (DFS QM-0004) [식품]
5. **구매계획 월별 12분해 + 클레임→채권 상계** [중]

## ⑤ 공통 플랫폼/엔진
1. **통합 알림센터** — 인앱+이메일+SMS+푸시 + 읽음/미읽음·딥링크·event_code + 템플릿 + 큐배치/재시도 (DFS 모바일 native.js + 메일배치) [상]
2. **위임전결규정 마스터 + 합의 병렬 + 결재의견 이력** — 금액·조직·문서유형별 전결권 산출 (참고시스템도 규정마스터 부재 → 추월) [상]
3. **데이터레벨 권한(법인·구매그룹) + SSO** (멀티테넌시 SYS_ID·SsoAuthHandler) [상]
4. **감사 2종** — 데이터변경(필드레벨 전후값 LOG_CHANGE) + 접근/다운로드/반출 (ESAULOG/ESATLOG/ESEXLOG) [상]
5. **배치/스케줄러 프레임워크** — Quartz 기반: 만료알림·자동마감·평가주기·SAP동기 (ApprovalScheduler·em-scheduler-mgt) [상]

## 권장 실행 순서 (테마 기반)

가장 레버리지 큰 횡단 테마부터:

**1차 — 워크플로우 폐루프(테마 A)**: 발주변경(①-1) + 입고변경·3-way(①-2) + 마감취소(②-1) — "변경/취소를 정식 상태머신으로". 감사·정합의 뿌리.
**2차 — 협력사 셀프서비스 + 폐루프(D+A)**: 협력사 실사 시정조치 폐루프(③-1) + CAPA 셀프회신(④-2) + 발주확약 포털(①-5).
**3차 — 식품 특화(테마 B)**: 의제매입세액(②-2) + 검사로트·특채·UD(④-1) + 원유검사(④-4) + AEO(③-2).
**4차 — 플랫폼 성숙도(테마 E)**: 통합 알림센터(⑤-1) + 전결규정(⑤-2) + 감사로그(⑤-4) + 스케줄러(⑤-5).
**5차 — 회계·지급 심화 + 공사용역(②+C)**: 어음·원천·상계(②-3) + AP에이징(②-4) + 공사용역 풀스택(①-4) + 소싱 심화(①-3) + 단가 시장가(③-4) + 전자계약(③-5).

> 모든 분석은 참고시스템 읽기전용 조회 기반. "추정" 표기 항목은 코드 직접근거 없이 실무표준으로 보완한 것. 정확한 컬럼 스펙은 구현 설계 시 해당 mapper 재확인.
