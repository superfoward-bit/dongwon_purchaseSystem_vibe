## B. 계약 (사용자 중점)

**DIPS:** 단가계약(pricecontract) + 일괄단가계약 + 품목별/요청부서별 단가현황 + 하도급계약 + 변경요청/승인/해지(mod_typ) + 변경이력.
**DFS(edoc 전자계약):**
- 계약유형: **표준거래계약(거래기본계약)**, 표준거래계약 갱신(버전), 기타거래계약(ETC), 거래약정(agreement), 마이그레이션계약.
- 계약서식(템플릿): 서식 등록/버전관리/확정, 첨부서식(필수/선택), 서식내용 조회. 계약항목 정의(입력창유형/데이터유형/필수/수정가능/맵핑컬럼).
- 전자서명(PKI): 본사+협력사 **이중 공인인증 서명**, 인증서 조회, 서명일자.
- 계약 미리보기·발송·오프라인계약·다운로드, 계약이력/차수(cntr_rev), 만료예정 관리.

**신규 시스템:** 단가계약(CT)만. TMP→REQ→ACTIVE→EXPIRE/TERMINATE/CANCEL, VAT, 결재연동, 품목 적용기간.

**갭/개선:**
- ❌ **표준거래계약(거래기본계약)** 없음 — 협력사와의 거래기본조건 계약. (계약의 근간)
- ❌ 전자계약/전자서명(PKI) 없음 — 계약서식·날인·협력사 서명. (실서명은 인증서 인프라 필요, 우선 서식/발송/내부전자결재 기반으로 대체 가능)
- ❌ 계약유형 다양성(기타계약/거래약정/일괄단가) 없음.
- 🔶 계약 변경/개정(Revision 차수) 관리 약함(CT_REV 컬럼만 존재, 변경 프로세스 없음).
- ❌ 만료예정 알림(D-day) 없음 — 이번 알림(CM_NOTICE)로 손쉽게 확장 가능.

## 종합 우선순위 제안 (다음 고도화 후보)

### 1순위 — 마스터/계약 근간 (사용자 중점 영역)
1. **협력사 마스터 확장** — 사업자/법인/ERP코드·대표자·연락처·은행·업종·지급조건·면허·이력 탭.
2. **협력사 신규등록 심사 워크플로우** — 신청→심사(등록평가)→승인→거래개시.
3. **표준거래계약(거래기본계약)** 도메인 신설 — 단가계약과 별개의 거래기본조건 계약 + 만료/갱신(차수).
4. **협력사 등급체계 정교화** — 평가팩터 가중치/세그먼트(SG)/등급행렬.

### 2순위 — 거래 실행 보강
5. **발주 고도화** — 발주유형(계약기반/일반/기본), 선금·기성·잔금, 발주서 출력.
6. **정산 보강** — 매입조정(가격/수량), 선급금/반제, 세금계산서/전표(또는 SAP연계로 이관).
7. **단가요소(factor)/단가그룹** — 원가구조 분해 단가.

### 3순위 — 품질/분석/공통
8. **품질 CAPA** — 부적합/개선대책, 품질성적서.
9. **품목 동적속성 + 등록요청 워크플로우**((식품)영양/규격 속성).
10. **실적 리포트** — GR실적/단가변동/장려금/클레임.
11. **공통** — 클레임 관리, 구매계획(purplan), 환율.

### 식품 특화(동원 맥락, 선택)
- 가락시장 시세연동 단가, HACCP/식품안전 API, 성장장려금, 불공정거래 신고.

### 이미 충분 (이번 고도화로 동등 이상)
- 전자결재(합의/참조/전결/위임/템플릿), 품목단가 구간관리, 갭정산 자동집계, RFP 평가입찰, 역경매 자동연장, 첨부/알림.
